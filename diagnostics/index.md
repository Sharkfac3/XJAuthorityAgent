# Diagnostics Index

## Purpose
Route symptom-first troubleshooting and fault isolation content.

## Use this branch when
The main question is what is wrong, what to test next, or how to verify or rule out likely faults.

## First-hop loading path
- Engine running or control-system complaint → [`engine/index.md`](engine/index.md)
- Fuel-delivery complaint → [`fuel/index.md`](fuel/index.md)
- Ignition complaint → [`ignition/index.md`](ignition/index.md)
- Overheating or cooling complaint → [`cooling/index.md`](cooling/index.md)
- Battery, charging, or starting complaint → [`charging-and-starting/index.md`](charging-and-starting/index.md)
- Transmission behavior complaint → [`transmission/index.md`](transmission/index.md)
- Transfer-case complaint → [`transfer-case/index.md`](transfer-case/index.md)
- Driveline complaint → [`driveline/index.md`](driveline/index.md)
- Steering or suspension complaint → [`steering-and-suspension/index.md`](steering-and-suspension/index.md)
- Brake complaint → [`brakes/index.md`](brakes/index.md)
- Body electrical complaint → [`body-electrical/index.md`](body-electrical/index.md)
- Body or interior complaint → [`body-and-interior/index.md`](body-and-interior/index.md)

## What belongs here
- symptom trees
- test sequences
- fault-isolation notes
- pre-repair and post-repair verification paths

## What does not belong here
- full service or installation procedures
- stock-only platform reference with no symptom context
- buying-only part recommendations
- repo governance
