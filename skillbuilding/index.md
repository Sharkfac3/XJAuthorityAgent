# Skillbuilding Index

Use this folder when the task is about **improving the repository**, not answering a Jeep technical question directly.

## When to use skillbuilding
Use `skillbuilding/` when you need to:
- identify missing swap-critical information
- model likely high-risk unknowns before a full audit
- design research tasks for unknown areas
- convert rough notes into canonical repo updates
- prioritize what to write next
- review whether a topic is mature enough to trust

## Ground rules
- Load `AGENTS.md` first for repo routing context.
- Use likely-gap files as planning hypotheses, not proof of repo deficiency.
- Do not store canonical Jeep facts in `skillbuilding/`.
- Move validated facts into `vehicle/`, `swap/`, or `book/`.
- Keep unresolved items explicit with labels like `unverified`, `needs source`, or `year-range uncertain`.
