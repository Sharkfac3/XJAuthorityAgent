# Skill: Source Intake

Use this skill when you have rough material that needs to be converted into repo-ready knowledge.

Examples:
- service manual notes
- connector pinout notes
- forum research summary
- vendor sensor data
- personal bench-test notes

## Steps
1. Read `AGENTS.md`.
2. Read `skillbuilding/evidence-standards.md` and `skillbuilding/source-policy.md`.
3. Read `skillbuilding/agents/fact-normalizer.md`.
4. Read the destination canonical file(s) if known.
5. Normalize findings with `skillbuilding/templates/source-intake.md`.
6. Separate validated facts from assumptions, conflicts, and open questions.
7. Recommend exact file placements.

## Required labels
- validated
- provisional
- conflicting
- needs source
- needs validation

## Success condition
A future edit into `vehicle/`, `swap/`, or `book/` should be straightforward and should not hide uncertainty.
